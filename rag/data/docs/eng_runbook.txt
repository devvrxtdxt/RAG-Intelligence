Engineering Deployment Runbook — Internal Use Only

1. Pre-Deployment Checklist
Before initiating any production deployment:
- Verify all CI/CD pipeline checks pass (unit tests, integration tests, linting)
- Confirm the release branch has been approved by at least two reviewers
- Check the deployment calendar on Confluence for conflicts or freeze windows
- Notify the #deployments Slack channel with the deployment ticket ID and expected duration
- Ensure rollback scripts are tested and accessible in the ops/rollback/ directory

2. Deployment Procedure
Step 1: Tag the release in Git using semantic versioning (e.g., v2.4.1)
Step 2: Trigger the deployment pipeline via Jenkins (job: prod-deploy-main)
Step 3: Monitor the Grafana dashboard (board: prod-health) during rollout
Step 4: Run the post-deployment smoke test suite: ./scripts/smoke_test.sh --env=prod
Step 5: Verify health endpoints return 200 on all three application servers (app-01, app-02, app-03)
Step 6: Update the deployment log in Confluence with timestamp, version, and deployer name

3. Rollback Procedure
If any smoke test fails or error rates exceed 2% within 15 minutes of deployment:
Step 1: Run ./ops/rollback/rollback.sh --to-version=<previous_tag>
Step 2: Notify #incidents channel immediately
Step 3: Page the on-call engineer if rollback does not resolve within 10 minutes
Step 4: File a post-incident report within 24 hours using the PIR template

4. On-Call Escalation
- L1 (first 15 min): On-call engineer (see PagerDuty rotation)
- L2 (15-30 min): Engineering lead for the affected service
- L3 (30+ min): VP of Engineering and SRE team lead
- Critical severity (data loss, security breach): Immediate escalation to CTO

5. Maintenance Windows
Standard maintenance windows: Saturday 2:00 AM – 6:00 AM UTC
Emergency maintenance requires approval from the VP of Engineering and 2-hour advance notice to affected customers via status page.
